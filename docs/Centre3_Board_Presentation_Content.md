## Slide 5: Dynamic Approval Workflow Engine

**Heading:** Intelligent Routing Reduces Approval Time by 60%

**Key Points:**
- Configurable workflow conditions route requests based on site, zone, risk level, visitor type, and time of day
- Multi-stage approval supports L1 through L5 levels with automatic escalation for overdue tasks
- Shift-based routing ensures approvals reach available personnel, eliminating bottlenecks from out-of-office approvers
- Delegation system allows temporary approval authority transfer during vacations or business travel

## Slide 7: Access Credential Delivery

**Heading:** Three Methods to Grant Secure Facility Access

**Access Methods:**

1. **QR Code Access** - System generates unique QR codes containing encrypted visitor credentials, scannable at entry points for touchless check-in
2. **RFID Tag** - Temporary RFID tags issued to visitors for extended access periods, automatically deactivated at visit end
3. **Access Card** - Physical card assignment for recurring visitors or long-term contractors with configurable zone permissions

**Credential Delivery:** Visitors receive access credentials via Email, SMS, or WhatsApp directly from the platform

## Slide 8: Real-Time Security Operations

**Heading:** Proactive Monitoring Prevents Security Incidents

**Key Points:**
- Global overwatch dashboard displays all active visits, pending approvals, and security alerts across facilities
- Automated alerts trigger for door forcing, tailgating attempts, access outside approved hours, and credential misuse
- Alert severity classification (Critical, High, Medium, Low) enables prioritized response by security teams
- Complete audit trail captures every access event with timestamp, location, and associated approvals

**Metric:** Average alert response time reduced from 15 minutes to under 2 minutes with automated notifications

## Slide 9: Hierarchical Facility Structure

**Heading:** Granular Control Across Your Entire Infrastructure Portfolio

**Structure:**
- **Sites** - Individual data center facilities with unique security policies and operational hours
- **Zones** - Logical groupings within sites (e.g., Colocation Hall, Network Operations, Power Plant)
- **Areas** - Specific access points within zones with individual security levels and approval requirements

**Capability:** Each level supports independent access policies, approval workflows, and security configurations

## Slide 10: Enterprise User & Group Management

**Heading:** Simplified Administration Through Role-Based Access

**Key Points:**
- User roles define system permissions (Administrator, Approver, Requestor, Security Officer)
- Group hierarchy supports organizational structure (Internal Employees, External Vendors, Contractors)
- Group-level access policies automatically apply to all members, reducing individual configuration
- Approval roles separate workflow participation from system access permissions

**Administration Benefit:** Onboard new team members in minutes by assigning to appropriate groups
